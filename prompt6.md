## 整体布局策略

游戏界面是一个全屏的桌游俯视视角。玩家围坐在"桌子"四周，中央是市场。关键难点是：玩家数量 3-7 人不固定，需要动态分配位置。

### 玩家位置分配算法

自己固定在正下方。其他玩家按顺序分布在上方和两侧：

```
3人: 上1               → [上左, 上右]           自己在下
4人: 上2               → [上, 左, 右]    自己在下
5人: 左1, 上2, 右1     → [左, 中左, 中右, 右]
6人: 左1, 上3, 右1     → [左, 上左, 上中, 上右, 右]
7人: 左2, 上2, 右2     → [左下, 左上, 上左, 上右, 右上, 右下]
```

使用 CSS Grid 或绝对定位实现。具体位置可以根据实际效果调整，但原则是：视觉均匀分布、自己始终在最下方。

每个玩家（包括自己）有一个面板，内容相同，但自己的更大更详细。


具体要求：
- 头像：使用默认图标（可以用 Font Awesome 的 user 图标）
- 昵称：加粗显示
- 资金：💰 图标 + 数字，数字变化时可以闪烁提示
- 放置区栅格：2行×3列，固定顺序 🦒→🐶→🦢→🐙→🦛→🐘
  - 每格显示：动物 emoji + 放置数量
  - 数量为 0：灰色背景、灰色文字
  - 数量 > 0：正常底色
  - **持有反垄断标记**：该格加金色/橙色背景高亮 + 小盾牌图标
- 手牌数量：其他玩家显示 "手牌: N张"，自己不显示（自己有手牌区）
- 当前回合指示：轮到该玩家时，面板边框高亮（发光效果或加粗彩色边框）

自己的面板（底部）更大，额外显示：
- 不需要"手牌: N张"（因为下面有手牌区）
- 放置区栅格可以稍大一些

其他玩家的面板更紧凑，适配屏幕边缘位置。上方玩家的面板可以是水平紧凑布局。左右两侧的面板垂直紧凑。

## 数据渲染逻辑

### game.js — 游戏页面主逻辑

```javascript
// 页面加载时
// 1. 从 sessionStorage 获取 player_name, room_code
// 2. 连接 WebSocket，发送 join_room
// 3. 注册消息处理器

gameWS.on('game_state', (data) => {
    updateGameUI(data.state);
});

gameWS.on('your_turn', (data) => {
    showTurnIndicator(data.phase);
    if (data.phase === 'play') showActionBar();
    // 交互逻辑在后续 prompt 实现
});

gameWS.on('game_end', (data) => {
    // 占位，后续 prompt 实现
    console.log('Game ended', data);
});

function updateGameUI(state) {
    renderMarket(state.market, state.deck_remaining);
    renderAllPlayers(state.players, state.current_player_id);
    renderMyHand(state.my_hand);
    renderDrawCost(state.draw_cost);
    updateTurnIndicator(state);
}
```

### 渲染函数要求

每个渲染函数接收数据、更新 DOM：
- `renderMarket(market, deckRemaining)` — 更新中央市场所有卡牌
- `renderAllPlayers(players, currentPlayerId)` — 更新所有玩家面板位置和内容
- `renderMyHand(hand)` — 更新自己的手牌区
- `renderPlayerPanel(player, position, isMe, isCurrent)` — 渲染单个玩家面板
- `renderDrawCost(cost)` — 更新牌堆下方的费用显示

所有渲染函数应该是**幂等的**——用相同的数据调用两次，结果一样。不要增量更新 DOM，每次收到 game_state 就完整重新渲染（简单可靠，后续可优化）。

## 样式要求

- 全屏布局，无滚动条（100vh × 100vw）
- 桌面背景：深绿色或木纹色，体现桌游感
- 卡牌：白底、圆角(12px)、轻微阴影，类似实体卡牌
- 卡牌背面：使用深色底 + 重复的小图案或纯色 + 中心 logo
- 玩家面板：半透明深色背景、圆角、轻微模糊效果
- 当前回合玩家：面板有脉冲动画的亮色边框
- 过渡动画：卡牌出现/消失用 CSS transition（opacity + transform）
